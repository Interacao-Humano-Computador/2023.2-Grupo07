    Verificação dos Storyboards

    Introdução

    Após realizado o planejamento geral de como serão realizadas as verificações da etapa 4, neste artefato será produzido de fato a Verificação dos Storyboards.

    Dessa forma, inicialmente serão apresentados os objetivos dessa verificação, a metodologia utilizada para sua realização, os responsáveis por realizá-la e as respostas dos checklists apresentados no planejamento da etapa 4. Além disso, será desenvolvida uma análise e posterior levantamento dos problemas identificados após o respectivo checklist. Sendo assim, a partir das principais incoerências encontradas, será possível trazer sugestões e avaliar a necessidade de retrabalho.

    Objetivo

    O objetivo deste documento é realizar uma verificação do que tange o artefato de Storyboards.

    Metodologia

    Os resultados da análise deste artefato foram obtidos através da análise dos checklists de verificação preparados na página de planejamento. Dessa forma, o avaliador irá verificar o respectivo artefato do grupo e responder às questões apresentadas nos checklists, utilizando as opções "Sim", "Não" ou "Incompleto" para as respostas, além de também poder registrar comentários em cada questão analisada para detalhar alguma observação pertinente.

    Participantes envolvidos

    Conforme explicitado no tópico "Cronograma" do Planejamento da Verificação da Etapa 4, os participantes responsáveis do Grupo 7 e suas respectivas funções são:

    • Suzane Duarte, a qual será responsável por realizar a avaliação;
    • Luana Torres, a qual será responsável por realizar a revisão do artefato produzido pelo avaliador.

    Problemas encontrados

    A partir do preenchimento da tabela 1 e 2, a seguir serão realizadas as considerações acerca dos itens mencionados com "NÃO" ou "Incompleto".

    ID 1 - Tabela 1: O artefato possui introdução?

    O artefato não possui um texto introdutório para explicar o que são Storyboards, o que deve possuir um Storyboard e o por quê de utilizar este artefato no projeto.

    ID 2 - Tabela 1: O artefato possui uma bibliografia/referência bibliográfica?

    O artefato não possui nenhuma seção para referenciar possíveis referências bibliográficas.

    ID 3 - Tabela 1: O artefato possui um histórico de versões com o id e descrição das versões, data, autores e revisores padronizado?

    O artefato possui uma tabela seguindo os padrões utilizados no restante do projeto, no entanto, não existe um título para a seção a qual deveria estar essa tabela.

    ID 4 - Tabela 1: Todas as tabelas e imagens são chamadas no texto, possuem legenda e fonte?

    Nenhuma das imagens apresentadas são chamadas no texto, além do que em alguns casos a legenda da imagem não está seguindo os padrões utilizados durante o restante do projeto (Figura 1 e Figura 2)

    ID 7 - Tabela 2: Todos os Storyboards são LEGÍVEIS?

    Ao realizar a leitura, pelo menos a partir da qualidade das imagens anexadas, nem todos os Storyboards são completamente legíveis. É possível compreender o enredo da história, porém, em determinados Storyboards houve uma certa dificuldade para o entendimento de determinadas palavras.

    ID 8 - Tabela 2: Todos os Storyboards apresentam uma parte para descrição sobre cada quadro do storyboard?

    Apesar de não possuir carater obrigatório, como foi realizado a apresentação da descrição em alguns storyboards (Figura 1, Figura 2 e Figura 3), deveria-se padronizar todos os Storyboards apresentados. Isto é, tanto na Figura 1 quanto na Figura 5 também deveria-se apresentar uma descrição em formato textual ao lado de cada quadro do storyboard.

    Síntese dos dados encontrados

    Dessa forma, a partir das respostas coletas nos checklists acima foi possível sintetizar dados encontrados em formato de gráfico, conforme pode ser visto na Figura 1.

    Sugestões de Correções

    Por fim, a partir das informações coletas, apresentadas e analisadas acima, sugere-se realizar as seguintes correções:

    • Padronização da página de Storyboards;
    • Correção de erros já existentes na página de Storyboards;

    A avaliadora do artefato pontuou problemas nos Storyboards apresentados, porém, não sugere correções a serem realizadas nessa parte uma vez que as entrevistas utilizando os Storyboards já foram realizadas com os usuários. Dessa forma, a relevância dos comentários realizados nesses casos limitam-se apenas a serem críticas construtivas para possíveis projetos futuros.

    Retrabalho

    Para a realização do retrabalho é sugerido que os próprios autores do artefato verificado realizem a correção dos problemas apresentados seguindo a lista de sugestão apresentada no tópico anterior. Se necessário, outros membros do grupo 8 podem realizar as correções propostas, porém, deve-se informar a mudança realizada na Tabela 3 pois selecionou-se exatamente o responsável por essa página de acordo com a tabela do Histórico de Versões.

    O responsável do grupo 7 por essa verificação irá realizar uma revisão das correções feitas, verificando se as correções foram executadas com exatidão. Assim que todas as alterações forem implementadas, deve-se alterar o campo "Status" da Tabela 3 apresentada a seguir de "Em análise" para "Finalizado".
